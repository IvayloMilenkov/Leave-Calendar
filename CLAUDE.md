# Leave Calendar

A small team leave-tracking web app. Users log in, mark days as planned/approved leave on a calendar, and see their teammates' leave alongside their own.

## Stack

- **React 18** + **TypeScript** — frontend, no routing library
- **Vite** — dev server and build
- **Supabase** — Postgres database + auth (Google OAuth) + realtime subscriptions
- **Motion (Framer Motion v12)** — animations throughout

## Running locally

```bash
npm install
npm run dev
```

Requires a `.env` file with:
```
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

## Architecture

### Auth flow (`src/context/AuthContext.tsx`)
Google OAuth via Supabase. Three-stage render in `App.tsx`:
1. Not logged in → `LoginPage`
2. Logged in but no team → `TeamSetup`
3. Logged in with team → `CalendarView`

### State

Two separate state layers:

**`AppContext`** — local UI state, persisted to `localStorage` (`leave-calendar-v1`):
- `config.totalAllowanceDays` — user's annual leave allowance (default 20)
- `leaveDays` — map of `"YYYY-MM-DD"` → `'planned' | 'approved'` for the current user
- `ui` — current view year/month and view mode (`'month' | 'year'`)

**`TeamContext`** — server state from Supabase:
- `team` — current team row
- `members` — all team members with their display names and assigned colors
- `teamLeaveDays` — all leave days across the entire team
- Realtime subscriptions on `leave_days` and `team_members` tables keep state live

### Supabase schema (`src/lib/supabase.ts`)

| Table | Key columns |
|---|---|
| `teams` | `id`, `name`, `invite_code`, `owner_id` |
| `team_members` | `team_id`, `user_id`, `color`, `display_name` |
| `leave_days` | `id`, `user_id`, `team_id`, `date` (YYYY-MM-DD), `status` |

RPC: `find_team_by_invite_code(code)` — used when joining a team via invite link.

### Leave day toggle cycle

Click cycles: none → `planned` → `approved` → none. Both local state (`dispatch`) and Supabase (`syncLeaveDay`) are updated on each click. On first app load, local state is overwritten with the authoritative Supabase data.

### Team features
- Create a team or join via invite code / URL (`?join=<code>`)
- Members get auto-assigned colors from a palette of 8; owner can change their own
- Owner can remove members; any member can leave
- Invite code can be regenerated by the owner

### Calendar views

- **Month view** — `CalendarGrid` (mobile) or `MonthCarousel` (desktop, shows 3 months side by side)
- **Year view** — `YearView` renders 12 `MiniMonth` components
- Toggle between views in `CalendarNav`
- Weeks start on Monday; grid built by `buildCalendarGrid` in `calendarUtils.ts`

### Holidays (`bulgarianHolidays.ts`)
Bulgarian public holidays hard-coded (fixed dates + Orthodox Easter via Meeus algorithm). Holidays are blocked from being toggled. Sunday holidays shift to the following Monday.

### Team dots + tooltip
`DayCell` renders colored dots for teammates who have leave on that day. Hover (desktop) or long-press (mobile, 500 ms) shows a `DayTooltip` portal listing who has the day booked and their status.
